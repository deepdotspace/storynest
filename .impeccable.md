# storynest — design brief

This is the canonical design context for storynest. Every design and build
sub-agent reads this and treats it as locked. Visual decisions here are
non-negotiable for v1; everything else is open craft.

## Product in one line

A picture-book maker for parents. The product *is* a printed children's
book — soft paper, warm ink, illustration-first, text as quiet companion.

## Brand mood

Storybook nostalgia, but contemporary. Think:
- A new hardcover children's book on the shelf at an independent bookshop.
- Warm cream paper, lightly textured (subtle noise OK, no aggressive grain).
- Hand-set serif with confident proportions; everything sits on a generous
  baseline grid.
- Playful but never childish-cute — designed for the parent reading aloud,
  not for the toddler tapping a tablet game.

What it is NOT:
- Not a SaaS dashboard. No data-density, no card grids that look like Trello.
- Not a Canva clone. No multi-tool toolbar, no scrapbook stickers, no neon.
- Not "AI app" — no chat bubbles in product chrome, no "Powered by AI"
  language, no sparkle icons next to "Generate".

## Type system

- **Display**: EB Garamond — book titles, page headings, the reader's text
  bubble. Weight 500 for titles, 400 for body. Use italics for narration in
  the reader. Load from Google Fonts.
- **Sans (UI chrome)**: Work Sans 400 / 500. Used ONLY for buttons, menu
  items, badges, toasts — UI affordances, never content. Tight letter
  spacing (-0.01em).
- **Hand**: Caveat 500 — used sparingly for one accent (e.g. "by you, for
  Mila" on the cover, page numbers like a kid wrote them). At most one
  Caveat instance per visual frame. Load from Google Fonts.

Sizes — modular scale 1.2:
- 12 / 14 / 16 / 19 / 23 / 28 / 33 / 40 / 48

Reader's narration text is **23px EB Garamond italic** with a 1.55 line-height
on a 720px max-measure. This is the most important type setting in the app —
it's what the parent reads aloud.

## Color (OKLCH)

Light mode is the only mode for v1. Children's books are not dark.

Tokens:
```
--paper:      oklch(0.975 0.013 84)    /* cream paper background */
--paper-deep: oklch(0.94 0.022 84)     /* slightly darker for card lift */
--ink:        oklch(0.22 0.035 265)    /* primary text — deep ink-navy */
--ink-soft:   oklch(0.45 0.030 265)    /* secondary text */
--ink-mute:   oklch(0.62 0.020 265)    /* tertiary / hint text */
--rule:       oklch(0.88 0.018 84)     /* hairlines, dividers */
--marigold:   oklch(0.78 0.155 72)     /* primary accent — buttons, links */
--marigold-d: oklch(0.65 0.155 60)     /* marigold pressed / dark variant */
--rose:       oklch(0.72 0.135 22)     /* secondary accent — error, hearts */
--sage:       oklch(0.72 0.075 150)    /* success state, "ready" badge */
```

Rules:
- Body text is `--ink` on `--paper`. Never lower the contrast.
- Primary buttons: solid `--marigold` background, `--paper` text, slight
  drop shadow (paper-on-paper feel).
- "Ready" badges: `--sage` filled pill. "Generating" badges: `--marigold`
  filled pill with a subtle pulse. "Failed": `--rose` outline pill.
- Cards lift via `--paper-deep` background + 1px `--rule` border + a 2-3px
  soft shadow tinted with `--ink` at 6% alpha. NO heavy shadows, NO neon
  rings on focus.

## Layout

- **Container**: 1120px max for library/index; **720px max-measure** for the
  reader's narration.
- **Spacing**: 8px base. Section padding 56–80px on desktop.
- **Cover thumbnails** (library): 3:4 portrait, 240×320, rounded 8px,
  paper-deep frame.
- **Reader**: full-viewport. Image takes the upper 60–70% on portrait
  viewports, full width on landscape. Text bubble is a centered floating
  callout, 720px max, sitting on top of the lower image edge.

## Components

A short list of patterns that recur:

- **Paperbook card**: rounded 8px, `--paper-deep` bg, hairline `--rule`
  border, tiny shadow. Cover art bleeds to the corners; title + author
  pair sits below in EB Garamond 19 + Caveat 14 (the byline is the Caveat
  accent).
- **Pill badge**: 11px Work Sans 500 uppercase tracking 0.05em. Sage /
  marigold / rose / ink-mute variants. No icons inside badges.
- **Primary button**: 14px Work Sans 500. Marigold solid, paper text, 8px
  radius, 12×20 padding, micro-shadow. Hover lowers shadow + shifts to
  `--marigold-d`.
- **Text bubble (reader)**: cream `--paper` at 92% alpha, 1px `--rule`
  border, 16px radius, 28-40px padding. Italics EB Garamond.
- **Form fields**: minimal — just a hairline bottom rule that thickens to
  `--marigold` on focus. No filled inputs, no shadow boxes. Labels are
  Work Sans 12 uppercase tracking 0.05em sitting above the field.

## Reader interaction principles

The reader is the soul of the app. Three rules:
1. **Quiet chrome.** Prev/next arrows are 28px circular `--paper-deep`
   buttons with a hairline border, faded to 60% opacity, full opacity on
   hover. Floating bottom corners.
2. **One audio control.** Single mute toggle, top-right, 32×32, otherwise
   audio plays automatically when a page becomes the current page.
3. **Text optional.** A tiny "hide text" toggle next to the mute icon lets
   the parent show only the picture (some pages don't need narration).

## What to NEVER do

- No purple-to-blue gradients. No gradient text. No glassmorphism.
- No "AI" / "Magic" / "Sparkle" wording in chrome.
- No exclamation marks in product chrome ("Story ready!" → "Your story is ready").
- No emojis in product chrome (the kid can see them in narration text if the
  story uses them, but never in the app shell).
- No `<select>` browser-default. Use a custom popover-based one.
- No `window.confirm` / `window.alert` / `window.prompt`. Use the SDK toast +
  dialog primitives, or build the patterns in-line where needed.
- No filled-shadow drop-shadow-2xl card piles. Cards are paper, not chits.
- Never use Inter, Fraunces, Newsreader, DM Sans, Plus Jakarta, Syne, IBM
  Plex, Outfit, Instrument, Cormorant, Crimson, Playfair, Space Grotesk.
  EB Garamond + Work Sans + Caveat are the entire type system.

## Tone of voice in product chrome

- "Library", "Read", "Create story", "Generating illustrations", "Your story
  is ready"
- NOT "Magic library", "Tap to read!", "AI is making your story!"
- Empty state on library: "No stories yet. Make your first one — it takes
  about a minute."

## Tagline (used on landing + signin)

"Bedtime stories, made for one little reader."

(EB Garamond italic, 28px, on cream paper, with a Caveat "for [child's name]"
underneath as the playful flourish.)
