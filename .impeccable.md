# storynest — design brief v2 (the colorful redesign)

Replaces the v1 "cream paper" brief. Every design and build sub-agent
reads this and treats it as locked. Visual decisions here are
non-negotiable.

## Identity in one line

storynest — a cozy world for little readers. A children's storybook app
that feels like a beloved picture-book publisher, not a SaaS tool.
Picture **Lingokids meets Yoto meets a hand-drawn indie children's
press** — not a yellow paper book, not a corporate dashboard.

## What we are NOT

- ❌ Yellow / cream paper book aesthetic (the v1 — looked old, sepia, dull).
- ❌ Single-accent minimal SaaS chrome (boring, doesn't read as a kids' brand).
- ❌ AI-design tells: purple-to-blue gradients, glow-on-dark, glassmorphism, sparkle icons next to "Generate", gradient text on a dark background.
- ❌ Neon, edgy, gamified ("UNLOCK!", "BOOST!", "LEVEL UP!") — kids' brand voice is warm and quiet, not arcade.

## What we ARE

- ✅ Multi-color, cheerful, confident. Five accent hues all sit together.
- ✅ Sticker / picture-book physicality: hard-offset shadows (no blur), thick ink outlines on illustrations, generously rounded corners.
- ✅ Custom illustrated character (**Hootie** the owl — see `src/components/mascots/Hootie.tsx`) appears on landing, empty states, and the create flow. Decorative SVG shapes (clouds, stars, books, hearts, moons) scattered as page accents.
- ✅ Background is clean cool-white (NOT cream). Color comes from the cards, mascots, and accents — not the page.
- ✅ Big radii on cards (20-24px), pill buttons (full-rounded), generous padding.
- ✅ Friendly type pairing: Fredoka (display, chunky-rounded) + Nunito (body, clean rounded sans) + Caveat (occasional hand accent).

## Color (OKLCH — defined in `src/themes.css` under `[data-theme="storynest"]`)

```
/* Surface */
--storynest-bg:        oklch(0.985 0.008 240)   /* cool clean white page */
--storynest-card:      oklch(1 0 0)             /* pure white card */
--storynest-card-soft: oklch(0.97 0.01 240)     /* slightly tinted alt card */
--storynest-rule:      oklch(0.91 0.015 240)    /* hairline divider */

/* Ink */
--storynest-ink:       oklch(0.22 0.04 265)     /* primary text — deep ink */
--storynest-ink-soft:  oklch(0.40 0.04 265)
--storynest-ink-mute:  oklch(0.58 0.03 265)

/* Accents (multi-color brand palette) */
--storynest-sky:        oklch(0.72 0.15 235)    /* PRIMARY — buttons, links */
--storynest-sky-deep:   oklch(0.55 0.16 235)    /* pressed / shadow underlay */
--storynest-sun:        oklch(0.85 0.16 92)     /* SECONDARY — highlights, badges */
--storynest-sun-deep:   oklch(0.72 0.16 78)
--storynest-coral:      oklch(0.74 0.16 30)     /* WARM ACCENT — hearts, hover */
--storynest-coral-deep: oklch(0.62 0.18 28)
--storynest-mint:       oklch(0.78 0.12 165)    /* SUCCESS — ready, complete */
--storynest-mint-deep:  oklch(0.62 0.14 162)
--storynest-lavender:   oklch(0.75 0.12 295)    /* TERTIARY — pro tier, magic moments */
--storynest-lavender-deep: oklch(0.58 0.14 295)
--storynest-bubblegum:  oklch(0.78 0.16 350)    /* PLAY — fun accent */
```

**Rules:**
- Primary CTA uses **sky**. Secondary CTA uses **sun**. Destructive uses **coral**. Success/ready uses **mint**. Pro/magic uses **lavender**.
- A given component should use no more than 2 accent colors at once (sky + sun is the most common pair).
- Background of the page is always `--storynest-bg`. Cards lift on `--storynest-card` (pure white) — the cool-white background makes the white cards pop.
- NEVER use the old `--storynest-paper` / `--storynest-paper-deep` / `--storynest-marigold` tokens. They are deprecated. The CSS file still defines them for legacy code but new work uses the new tokens.

## Type system

```
--storynest-font-display: 'Fredoka', system-ui, sans-serif
--storynest-font-body:    'Nunito', system-ui, sans-serif
--storynest-font-hand:    'Caveat', cursive
```

- **Fredoka** (Google Fonts) — chunky rounded display. Weights 400/500/600/700. Use for titles, hero text, big numbers, primary headings.
- **Nunito** (Google Fonts) — rounded sans-serif. Weights 400/500/700. Use for body text, buttons, UI chrome, narration text in the reader (KEEP NUNITO for narration — the previous EB-Garamond italic narration is REPLACED).
- **Caveat** — sparing, for "by you, for Mila"-style flourishes only. Max one Caveat instance per visual frame.

**Banned forever:** Inter, Fraunces, Newsreader, DM Sans, Plus Jakarta, Syne, IBM Plex, Outfit, Instrument, Cormorant, Crimson, Playfair, Space Grotesk, **EB Garamond** (was used in v1 — drop it). Work Sans is also retired in v2.

**Type scale (modular 1.25):**
12 / 14 / 16 / 20 / 25 / 32 / 40 / 48 / 56 / 72

**Reader narration text:** 24px Nunito 500, line-height 1.55, max-measure 680px. (Replaces the v1 EB-Garamond italic.)

## Shape language

- **Cards:** radius 20-24px, white bg, hairline `--storynest-rule` border, **hard offset shadow**: `4px 4px 0 0 var(--storynest-ink)` at 8% alpha (use `oklch(0.22 0.04 265 / 0.08)` form). On hover lift the shadow to `6px 6px 0 0 ...` at 10%. NO blurred soft shadows.
- **Buttons:** **pill / full-rounded**. Primary = sky bg, white text, 6px hard offset shadow in sky-deep. Secondary = white bg, ink-soft text, hairline border, no shadow. On press buttons collapse: shadow goes to `0 0 0 0` and the button translates down/right by the offset (gives the "sticker pressed" feel — same as crane).
- **Pills / badges:** Small full-rounded, 11px uppercase tracking 0.05em, single accent fill or hairline-bordered.
- **Inputs:** 16px radius, white bg, 1.5px `--storynest-rule` border that thickens to `--storynest-sky` on focus (3px). NO underline-only inputs — the v1 underline style is dropped.
- **Mascot frame:** when Hootie appears at large size, frame the surrounding region with `--storynest-card-soft` and a 1px `--storynest-rule` ring (gives him a "diorama" feel).

## Mascot — Hootie

Hootie is a small, friendly owl. Three variants live in `src/components/mascots/Hootie.tsx`:
- `reading` — perched on a stack of books, big eyes closed in concentration. Used on the landing hero and the explore empty state.
- `waving` — wing raised, eyes open and friendly. Used on signup / library empty state.
- `sleeping` — eyes closed, tiny zzz's. Used on the reader's "audio muted" state and signed-out states.

Anatomy locks:
- Body: ink-outlined teardrop, fill = `--storynest-lavender`.
- Belly: cream oval (use `oklch(0.96 0.03 70)`).
- Eyes: large round white, ink iris, white catch-light.
- Beak: triangle, fill = `--storynest-sun-deep`.
- Branch / books under feet: `--storynest-mint` for the branch, hand-drawn book stack in coral/sky/sun.
- Outline weight: 3px on outer silhouette, 1.5px on inner details.
- Always shipped as inline SVG, never a raster image.

## Decorative shapes (illustrated UI)

In `src/components/decor/` — small inline SVGs that scatter across pages as visual decoration. Used sparingly, NEVER as functional icons:

- `Cloud` (sky-tinted, soft)
- `Star` (sun-tinted, 5-point with hard outline)
- `Heart` (coral)
- `Moon` (lavender crescent with face)
- `OpenBook` (small stack/spread)
- `Rainbow` (arc, multi-color)
- `Sparkle` (mint, four-pointed — NOT the AI sparkle, used quietly for "delight" moments)

Rules:
- Decorative shapes are positioned absolutely, behind interactive elements (`z-index: 0` or `-1`), at 80-90% opacity.
- A page surface uses 3-6 decorative shapes max — enough to feel "alive," not chaotic.
- Decorative shapes never overlap text. They live in margins, corners, and behind hero blocks.

## Voice and copy

- "Make a story", "Read aloud", "Your shelf", "Tap to flip"
- NOT "Magic library", "AI-generated", "Powered by AI", "Unlock", "Discover", emojis in product chrome
- A touch of warmth allowed: "Hi there, little reader" (signed-in greeting in nav), "Off to dreamland" (book deleted toast).

## Banned

Same as v1, plus:
- No yellow / cream / sepia tinted backgrounds.
- No EB Garamond, Work Sans (v1's fonts — retired).
- No subtle paper grain on the body (was in v1 — dropped).
- No filled drop-shadow-2xl card piles.
